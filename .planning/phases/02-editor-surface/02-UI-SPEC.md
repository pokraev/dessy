## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|

| sm | 8px | Padding within tool buttons, section internal padding, panel header horizontal padding |

## Typography

All sizes in pixels. UI chrome only — canvas content typography is user-controlled.

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Body | 13px | 400 | 1.4 | Inter, sans-serif | Panel labels, layer names, property field labels |
| Label small | 11px | 400 | 1.3 | Inter, sans-serif | Section sub-labels, unit suffixes (mm, °, %), tooltip text |
| Heading | 11px | 600 | 1.3 | Inter, sans-serif | Section headers (POSITION, FILL, SHADOW, etc.), panel tab labels |
| Numeric | 13px | 400 | 1.0 | JetBrains Mono, monospace | All numeric inputs (X, Y, W, H, rotation, opacity, font size) |

Rules:
- All section headers use uppercase tracking (letter-spacing: 0.08em)
- Layer names truncate with ellipsis at panel width minus icons
- "Mixed" indicator for multi-selection: italic, color-text-secondary (#888888)
- Font weight is exactly 2 values: 400 (regular) and 600 (semibold)

- Tab bar: 36px height, `bg-surface`, 3 equal-width tabs
- Active tab: bottom 2px border `#6366f1`, text `#f5f5f5` weight 600

- Type icons: `Type` (text), `Image` (image), `Square` (shape), `RectangleHorizontal` (colorBlock)
- Eye icon: `Eye` when visible / `EyeOff` when hidden; color `#888` normally, `#f5f5f5` on hover
- Lock icon: `Lock` when locked / `Unlock` when unlocked; color `#888` normally

- Row hover: `bg-surface-raised` at 60% opacity

**PageThumbnail** (inside PagesPanel.tsx)

- Right-click context menu: reuse existing ContextMenu component

### Properties Panel

**SectionHeader** (existing, reuse unchanged)
- 11px uppercase, weight 600, letter-spacing 0.08em, color `#888888`
- Chevron for collapse/expand
- Toggle switch for shadow section on/off state

**NumberInput** — reusable across all position/size/rotation/opacity inputs
- Height: 28px, padding: 4px 8px
- Background: `#1e1e1e`, border: `1px solid #2a2a2a`
- Border radius: 4px
- Font: JetBrains Mono 13px weight 400
- Focus: border `1px solid #6366f1`
- Suffix label (mm, °, %) positioned absolute right-8px, 11px, color `#888`

**AspectLockButton** — icon toggle between W and H inputs
- 20×20px, icon `Link` (locked) / `Link2Off` (unlocked) from lucide-react

**Gradient editor for fill** (Claude's discretion — spec below)
- Two-stop linear gradient only for v1
- Direction picker: 8 angle presets as icon buttons (45° increments) in a 2×4 grid
- Stop 1 color + Stop 2 color: each a 24×24px swatch that opens ColorPicker popover

**Corner radius input**
- Single NumberInput labeled "Radius" with unit "px" (not mm — pixel radius matches Fabric.js rx/ry)
- For shapes with all-corners-equal radius only (v1)

**Multi-selection indicator**
- Shown when 2+ objects selected
- Replace numeric value with em-dash ("—") and italic color `#888`
- All inputs are disabled in mixed state except opacity

### Color Picker Popover

**ColorPicker.tsx** — single reusable popover component
- Trigger: colored 24×24px swatch button, border `1px solid #2a2a2a`, border-radius 4px
- Popover: 240px wide, `bg-surface-raised` (`#1e1e1e`), border `1px solid #2a2a2a`, border-radius 8px, box-shadow `0 8px 24px rgba(0,0,0,0.6)`
- Popover opens upward if trigger is in bottom half of viewport, downward otherwise
- Closes on click-outside (use Framer Motion `AnimatePresence` for open/close fade 150ms)

Popover internal layout (top to bottom):
1. `react-colorful` HexColorPicker — full width (240px), height 160px
2. HEX / RGB / HSL tab switcher — 3 text tabs, 28px height, 11px labels
3. Active input row: HEX = single text input; RGB = 3 number inputs (R/G/B 0-255); HSL = 3 number inputs (H 0-360, S/L 0-100%)
4. Eyedropper button row: full-width 28px button, `Pipette` lucide icon + label "Pick from screen", disabled with tooltip "Not supported in this browser" when EyeDropper API unavailable
5. "Recently used" label 11px + 8px gap + row of up to 12 swatches (20×20px each, 4px gap)
6. "Brand colors" label 11px + 8px gap + row of up to 10 swatches (24×24px each, 4px gap) + "+" button to open swatch editor

**Predefined palettes** (Claude's discretion resolution):
- Housed inside ColorPicker popover as a collapsible "Palettes" section at the bottom
- 20+ palette names displayed as labeled rows; clicking a palette name reveals 5-8 color swatches inline
- Clicking a palette swatch copies that HEX into the color picker without closing the popover
- No separate panel needed

**Palette generator** (STYL-02 — Claude's discretion resolution):
- Housed in the Style section of PropertiesPanel (nothing selected state)
- "Generate palette" button opens an inline section: user picks a base color via swatch (opens ColorPicker), clicks "Generate", system produces 5-color complementary palette using HSL math (triadic + split-complementary)
- Generated swatches appear as a row; user can "Add all to brand colors" or click individual swatches to add

### Typography Section

**GoogleFontsDropdown**
- Trigger: current font name in 13px rendered in its own typeface, full-width button, height 32px, `bg-surface-raised`, `border border-border`

- Divider + "All fonts" section header followed by remaining fonts

- Fonts loaded on-demand: load batch of 20 when user scrolls within 5 items of list bottom (FontFace API)
- Loading indicator: single animated dot after last loaded font row

**Typography preset buttons**
- 5 buttons in a 2-3 layout (2 top row: Headline, Subhead; 3 bottom row: Body, Caption, CTA)
- Each: height 28px, `bg-surface-raised`, border `1px solid #2a2a2a`, border-radius 4px, 11px label
- Hover: border `1px solid #6366f1`
- Active (currently applied to selection): border `1px solid #6366f1`, background `rgba(99,102,241,0.15)`

**Text alignment buttons**
- 4-button group: AlignLeft, AlignCenter, AlignRight, AlignJustify

- Active button: `bg-accent` fill with icon color `#f5f5f5`

**Text transform buttons**
- 3-button group: "AA" (none), "aa" (lowercase), "AA+" (uppercase)... use text labels not icons
- Same size and styling as alignment buttons

## Interaction Contracts

### Panels
- Left and right panels animate open/close: width 0→280px (left) or 0→320px (right), `duration: 0.2s easeOut` using Framer Motion (established in EditorLayout.tsx)
- Panel sections collapse/expand with `height: auto` animation, `duration: 0.15s easeOut`
- Shadow section expands only when shadow toggle is on

### Layer Operations
- Click: selects canvas element, auto-switches active tool to `select`
- Double-click name: activates inline rename (contenteditable), blur or Enter to commit
- Eye icon click: toggle visibility immediately, canvas re-render
- Lock icon click: toggle lock state immediately
- Drag-to-reorder: @dnd-kit/sortable; visual gap indicator shows drop position; canvas.moveTo() called on drop to sync z-index

### Page Operations
- Click thumbnail: triggerSwitchPage(index)
- Right-click thumbnail: ContextMenu with items: "Duplicate Page", "Delete Page" (destructive — see copywriting)
- Drag-to-reorder: @dnd-kit/sortable; page order updates projectStore and sessionStorage keys
- "+ Add Page" button: appends blank page, switches to it

### Property Inputs
- All number inputs: on blur OR Enter → apply change to canvas object; Escape → revert to last committed value
- Arrow up/down keys on focused number input: increment/decrement by 1 (Shift+arrow = 10)
- All property changes: 300ms debounce before recording history snapshot (undo grouping)
- Opacity: range slider (0-100) + NumberInput side-by-side, synchronized

### Color Picker
- Opening: renders in portal at `z-index: 200` above all panels
- Color changes: update canvas object in real-time (no debounce for color — immediate visual feedback)
- Brand swatch click: instantly applies that swatch's color value AND stores swatchId on object
- Eyedropper: activates native EyeDropper API; canvas pointer becomes crosshair; picked color populates HEX input
