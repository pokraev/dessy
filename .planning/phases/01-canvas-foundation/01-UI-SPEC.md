# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Canvas Foundation. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — Tailwind CSS 4 utility-first, no component library |
| Preset | not applicable |
| Component library | none (Radix primitives may be added in Phase 2 when full panel system is built) |
| Icon library | Lucide React (standard for React/Tailwind ecosystems; icon names match tool toolbar vocabulary: `MousePointer2`, `Type`, `Square`, `Circle`, `Minus`, `Image`, `Hand`) |
| Font | Inter (UI labels, buttons, panels) + JetBrains Mono (measurement values, coordinate inputs) — loaded from Google Fonts |

> Source: CLAUDE.md Design System section. No `components.json` present (greenfield project — no shadcn init needed at this phase; shadcn gate: not applicable for Phase 1 which builds primitives from scratch).

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gap within toolbar buttons, inline icon-label gap |
| sm | 8px | Compact element spacing, gap between adjacent toolbar icons |
| md | 16px | Panel internal padding (all four sides), form row gaps |
| lg | 24px | Section gap within panels (e.g. gap between "File" and "Edit" header groups) |
| xl | 32px | Modal internal padding |
| 2xl | 48px | Vertical breathing room around empty-state illustrations |
| 3xl | 64px | Not used in Phase 1 (no page-level layouts) |

Exceptions:
- Input height: 32px (from CLAUDE.md — aligns with 8-point grid)
- Left panel width: 280px
- Right panel width: 320px
- Toolbar icon size: 16px (toolbar and panel icons, unified)
- Touch/click target minimum: 32px (all interactive elements hit area — aligns with 8-point grid, respects dense editor paradigm; 44px AAA would force toolbar too wide)
- Border radius: 8px (inputs, buttons, cards, popovers), 12px (modals, keyboard shortcuts overlay)

> Source: CLAUDE.md Spacing section. Revised: icon sizes unified to 16px (was 18px toolbar / 14px panel — both non-multiples of 4); border-radius for inputs/buttons corrected to 8px (was 6px — non-multiple of 4, now unified with cards/popovers).

## Typography

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Panel heading | 11px | 500 (medium) | 1.0 (single line, uppercase) | Inter | Section labels in left/right panels — ALL CAPS, wide letter-spacing (0.08em) |
| Label | 12px | 400 (regular) | 1.4 | Inter | Input labels, secondary info, tool tooltips |
| Value / Body | 13px | 400 (regular) | 1.5 | Inter | Button text, panel values, toast copy, shortcut overlay descriptions |
| Measurement | 13px | 400 (regular) | 1.0 | JetBrains Mono | mm/px coordinate values in bottom bar, ruler tick labels, position/size inputs |

Weights used: 400 (regular) and 500 (medium). No bold (700) in the UI shell — bold is reserved for canvas content authored by the user, not the editor chrome.

**Deliberate tight scale note:** The 11/12/13px scale is intentional for editor chrome. This is a dense design tool UI (comparable to Figma, Affinity Designer) where panel space is precious. These sizes are NOT used for user-authored canvas content — they apply exclusively to the editor shell chrome. Confirmed per CLAUDE.md typography guidelines.

> Source: CLAUDE.md Typography section. Sizes locked: 11, 12, 13px (plus mono 13px for measurements). Three sizes, two weights — within contract limits.

## Color

All colors are for the dark-theme editor UI. The canvas document itself has a white (#FFFFFF) background (user-authored content, not UI chrome).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #0a0a0a | App background, canvas pasteboard behind the document, full-screen editor base |
| Secondary (30%) | #141414 (surface) / #1e1e1e (surface raised) | Left panel, right panel, header, bottom bar, modal backdrops, context menu background |
| Accent (10%) | #6366f1 (indigo) | See reserved list below |
| Accent hover | #818cf8 | Hover state on any accent-colored interactive element |
| Border | #2a2a2a | All panel dividers, input borders, separator lines |
| Text primary | #f5f5f5 | All readable body text, input values, button labels |
| Text secondary | #888888 | Muted labels, placeholder text, disabled states, ruler tick labels |
| Destructive | #ef4444 | Delete actions only — see copywriting contract |
| Success | #22c55e | Auto-save toast, export success toast |
| Warning | #eab308 | localStorage quota warning toast |

Accent (#6366f1) reserved exclusively for:
1. Active tool button in the left toolbar (background highlight)
2. Selected element's resize/rotate handles on the Fabric.js canvas
3. Focused input ring (outline on focused text inputs and numeric inputs)
4. Primary action buttons: "Save Project" in header, "Export JSON" in export trigger
5. Keyboard shortcut overlay trigger button (bottom-right `?` badge)

Accent is NOT used for: hover states on passive labels, panel backgrounds, borders, ruler ticks, grid lines, guides, or any decorative element.

Canvas guide colors (not app UI — these render inside the Fabric.js canvas overlay):
- Bleed zone overlay: rgba(239, 68, 68, 0.12) fill + #ef4444 1px border (red-tinted, semi-transparent)
- Margin guides: #06b6d4 (cyan) at 1px, 50% opacity
- Snap / smart guides: #ec4899 (magenta) at 1px, full opacity — appear only during drag
- Grid (when enabled): dots at #2a2a2a, 1px diameter, spacing matches mm grid at current zoom
- Fold lines: #6366f1 (accent indigo), 1px dashed, dash pattern 4px on / 4px off
- Rulers: #141414 background, #2a2a2a tick marks, #888888 tick labels (12px mono)

> Source: CLAUDE.md Colors section. Guide colors: bleed from CONTEXT.md decision (red-tinted overlay). Margin guide cyan selected from CONTEXT.md discretion ("cyan or magenta are both fine" — cyan chosen to distinguish from magenta snap guides). Grid dots chosen over lines (discretion decision — dots are less visually dominant on a dense canvas). Snap guide magenta is industry standard per CONTEXT.md.

## Interaction Contracts

These are Phase 1-specific behavioral specs that the executor must implement exactly.

### Canvas Pasteboard
- Pattern: 10px × 10px checkerboard, alternating #141414 and #1a1a1a (very subtle, near-invisible — transparency metaphor)
- The document page itself: white (#FFFFFF) with 4px box-shadow rgba(0,0,0,0.6) to separate from pasteboard

### Rulers
- Height/width: 20px
- Background: #141414
- Tick marks: #2a2a2a (major), #1e1e1e (minor)
- Tick labels: 9px JetBrains Mono, #888888, update as canvas pans/zooms
- Corner square (intersection): #0a0a0a fill, no label

### Toolbar (left panel, tool selection)
- Icon button size: 32px × 32px (hit area), 16px icon inside
- Inactive state: transparent background, #888888 icon
- Hover state: #1e1e1e background, #f5f5f5 icon
- Active state: #6366f1 background, #f5f5f5 icon
- Tooltip: appears after 600ms hover delay, positioned to the right of toolbar, 12px Inter, #141414 background, #f5f5f5 text, 4px border-radius, shows tool name + keyboard shortcut (e.g. "Select V")
- `aria-label` declarations for icon-only toolbar buttons (required — no visible text label):
  - Select tool: `aria-label="Select tool (V)"`
  - Text tool: `aria-label="Text tool (T)"`
  - Rectangle tool: `aria-label="Rectangle tool (R)"`
  - Circle tool: `aria-label="Circle tool (C)"`
  - Line tool: `aria-label="Line tool (L)"`
  - Image tool: `aria-label="Image tool (I)"`
  - Hand / Pan tool: `aria-label="Pan tool (H)"`

### Context Menu (right-click)
- Background: #1e1e1e, 8px border-radius, 1px border #2a2a2a
- Item height: 32px, 16px horizontal padding
- Item font: 13px Inter, #f5f5f5
- Destructive item (Delete): #ef4444 text, no background change
- Separator: 1px #2a2a2a
- Keyboard shortcut hint: aligned right, 12px, #888888
- Shadow: 0 8px 24px rgba(0,0,0,0.5)

### Header
- Height: 48px
- Background: #141414
- Bottom border: 1px #2a2a2a
- Project name: 13px Inter, 500 weight, #f5f5f5, editable on click (inline input, no separate modal)
- Button group (save, undo, redo, export): 32px height, 8px horizontal padding, 8px border-radius
  - Save Project: accent (#6366f1) fill button, #f5f5f5 text
  - Undo/Redo: ghost buttons, disabled state at #2a2a2a icon when no history available

### Bottom Bar
- Height: 36px
- Background: #141414
- Top border: 1px #2a2a2a
- Zoom slider: 160px wide, accent-colored track fill
- Zoom percentage: 13px JetBrains Mono, #f5f5f5, click to type custom value
- Page indicator: "Page 1 of 1", 12px Inter, #888888
- Grid toggle: icon button 28px × 28px, active state uses accent

### Collapsible Panels
- Animation: height 0 → auto via Framer Motion `AnimatePresence`, 200ms ease-out
- Panel section header: 32px height, 11px Inter uppercase 500 weight, #888888 text, chevron icon 16px right-aligned
- Expanded chevron rotates 180° on collapse, 150ms transition

### Snap Guides
- Threshold: 6px (midpoint of 5–8px industry range per CONTEXT.md discretion)
- Color: #ec4899 magenta, 1px solid
- Appear on `object:moving`, disappear on `mouse:up`
- Equal-spacing indicators: same magenta, dashed

### Undo/Redo
- Drag operations: one step per mousedown→mouseup (CONTEXT.md discretion: standard choice)
- Text editing: word-level grouping (CONTEXT.md discretion: word-level is standard)
- Property changes: debounced 300ms (locked in CONTEXT.md)
- History cap: 50 steps (CANV-07)

### Toast Notifications
- Position: bottom-right, 16px from edge
- Width: 320px max
- Background: #1e1e1e, 8px border-radius, 1px border #2a2a2a
- Leading icon: 16px, colored by type (success: #22c55e, error: #ef4444, info: #6366f1)
- Title: 13px Inter 500, #f5f5f5
- Body (optional): 12px Inter, #888888
- Auto-dismiss: 3s (success), 5s (error — user needs time to read)
- Entry animation: slide in from right + fade, 200ms ease-out

### Keyboard Shortcuts Overlay
- Trigger: `?` key (UXSH-06)
- Presentation: centered modal, 600px wide, #141414 background, 12px border-radius, 1px border #2a2a2a
- Backdrop: rgba(0,0,0,0.6)
- Sections: Tools, Canvas, Edit, File — each section has 11px uppercase heading
- Row: shortcut key(s) in 12px JetBrains Mono #888888 + description 13px Inter #f5f5f5
- Close: Escape key or clicking backdrop

### Image Frame Placeholder
- Empty image frame: #1e1e1e fill, 1px dashed #2a2a2a border
- X pattern: two diagonal lines, 1px, #2a2a2a — covers the full frame
- "Drop image" label: 12px Inter, #888888, centered in frame, only shown when frame is ≥ 60px on shorter side

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (header save) | "Save Project" |
| Primary CTA (JSON export) | "Export JSON" |
| Format picker modal heading | "New Leaflet" |
| Format picker CTA | "Create Leaflet" |
| Empty canvas first-open | "Your canvas is ready — select a tool to start designing" (shown as a subtle hint in the pasteboard area, disappears after first element is placed, never shown again) |
| Empty state (no element selected, right panel) | "Select an element to edit its properties" |
| Auto-save success toast | "Project saved" |
| Auto-save error toast (quota exceeded) | "Storage full. Export your project to free up space." |
| JSON export success toast | "Project exported" |
| JSON load success toast | "Project loaded" |
| JSON load error toast | "Could not load project. Make sure the file is a valid Leaflet Factory JSON." |
| Undo toast (shown at undo limit) | "Nothing left to undo" |
| Redo toast (shown at redo limit) | "Nothing left to redo" |
| Context menu — Delete item | "Delete" |
| Delete confirmation | No modal confirmation for single element delete — Delete key removes immediately (standard canvas behavior). Undo recovers it. |
| Context menu — Lock item | "Lock" / "Unlock" (toggle label) |
| Keyboard shortcuts overlay title | "Keyboard Shortcuts" |

Destructive actions in Phase 1:
- **Delete element**: via Delete key or context menu "Delete" — no confirmation modal; recoverable via undo (CANV-07). Destructive label color: #ef4444.
- **No other destructive actions in Phase 1** (clear history, delete project — those are Phase 4).

> Source: Requirements UXSH-02, UXSH-05, CANV-07, CANV-10, EXPO-03. Copy written to be direct, action-complete, and plain (no "Successfully" prefix — wastes characters). Primary CTA updated from "Save" to "Save Project" (verb + noun, matches "Project saved" toast copy).
