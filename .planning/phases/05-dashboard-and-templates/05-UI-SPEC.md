## Typography

All sizes use `Inter, sans-serif`. Two weights only: 400 (regular) and 600 (semibold).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | Card last-edited date, card page count, modal body copy, category tab labels |
| Label | 12px | 400 | 1.4 | Format badge text, card meta secondary info, template category tag, template card name and category |
| Heading | 16px | 600 | 1.3 | Project card title, modal section headers, dashboard section label ("Recent Projects") |
| Display | 28px | 600 | 1.2 | Dashboard hero title ("Dessy"), empty state heading |

Source: Confirmed from `WelcomeScreen.tsx` (28px/700 hero, 14px body â€” weight normalized to 600 semibold for consistency with rest of app) and `Header.tsx`.

Note: `WelcomeScreen` uses weight 700 for h1. Dashboard display heading uses 600 to stay within the declared 2-weight system. The visual difference is negligible at this size.

Note: Component inventory previously listed 11px and 13px sizes â€” these are mapped to the declared scale: 11px â†’ 12px (Label role), 13px â†’ 14px (Body role). No exceptions to the 4-size scale.

## Interaction Contracts

### Project card click
- Single click on card body (not 3-dot menu): call `openProject(id)` â†’ switches to editor view
- Click on 3-dot button: show inline dropdown menu, stop event propagation

### 3-dot menu
- Opens on click (not hover) â€” prevents accidental trigger
- Closes on: clicking outside (global mousedown listener), clicking any menu item, pressing Escape
- Only one menu open at a time

### Delete flow
- Click "Delete" in 3-dot menu â†’ inline confirmation replaces menu: "Delete '[name]'? This cannot be undone." + "Keep Project" (ghost) + "Delete" (red fill) buttons
- No separate modal for delete â€” inline in menu per existing header clear-canvas pattern
- On confirm: call `deleteProject(id)`, call `deleteThumbnail(id)`, refresh project list

### Duplicate flow
- Click "Duplicate" â†’ immediately creates "Copy of [name]", refreshes list, no confirmation needed

### New Leaflet flow
1. Click "New Leaflet" â†’ open `NewLeafletModal` on "Blank" tab
2. Click format card â†’ immediately creates project and calls `openProject(newId)`
3. For Custom format: clicking the Custom card reveals mm dimension inputs; requires both W and H before creating

### Template use flow
1. Click template card in gallery â†’ open template preview modal
2. Click "Use This Template" â†’ deep-copy JSON, assign new `crypto.randomUUID()` id, save via `saveProject()`, call `openProject(newId)`
3. No separate naming step â€” default name is the template name

### Logo click (editor â†’ dashboard)
- Clicking "Dessy" in editor header: trigger auto-save first, then call `goToDashboard()`

## State Handling

### Loading states
- Project list loads synchronously from localStorage (`listProjects()`) â€” no async spinner needed
- Thumbnail loads asynchronously from IndexedDB â€” card renders immediately with `#6366f1` placeholder, thumbnail replaces it when resolved
- Template gallery renders instantly from static imported JSON â€” no loading state

### Error states
- If `listProjects()` throws (corrupted localStorage): show error toast using existing `Toast.tsx` system; display empty state as fallback
- If `openProject()` fails (corrupted project JSON): show error toast "Could not open project. The data may be corrupted."
- If `duplicateProject()` fails: show error toast "Could not duplicate project. Try again."

### Empty state
- Condition: `listProjects()` returns empty array
- Show: `EmptyState` component (illustration + heading + body + CTA + 3 template suggestions)
- Template suggestions in empty state: first 3 entries from `TEMPLATES` array (index 0â€“2), one from Sale, Event, Restaurant
