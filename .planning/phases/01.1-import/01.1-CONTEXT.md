# Phase 01.1: AI Leaflet Generation - Context

**Gathered:** 2026-03-27
**Status:** Ready for planning
**Source:** PRD Express Path (.planning/phases/01.1-import/PRD.md)

<domain>
## Phase Boundary

This phase adds an AI-powered leaflet generation feature to the existing Fabric.js 7 editor. Users generate initial layouts from 3 input modes (text prompt, photo of existing leaflet, hand-drawn sketch), which deserialize directly into the existing canvas. This phase does NOT modify existing stores, canvas logic, or components — only extends with new modules.

</domain>

<decisions>
## Implementation Decisions

### AI Generation Modal
- New modal/dialog component with 3 tabs: Prompt, Photo, Sketch
- All tabs include: fold type selector (single/bi-fold/tri-fold/z-fold), optional logo upload, loading state, preview, "Load into Editor" button, "Regenerate" button
- Prompt tab: text description input + fold type picker
- Photo tab: upload photo of existing leaflet → AI extracts layout DNA (abstracted recreation, not copy)
- Sketch tab: upload hand-drawn sketch photo → AI interprets boxes/scribbles/blobs into structured layout
- Style modal consistently with existing editor dark theme (#0a0a0a bg, #6366f1 accent, Inter font)

### AI Service Layer
- Support both Claude API (Anthropic) and Google Gemini API as providers
- Provider-agnostic abstraction so either can be used or swapped
- AI outputs valid Fabric.js canvas JSON matching existing schema exactly
- Image/sketch modes use vision/multimodal capabilities of both providers
- Use Next.js API routes (not server actions) for AI calls — consistent with existing /api/ pattern in CLAUDE.md

### Panel-Aware Generation
- Single page: 1 canvas (A4 dimensions)
- Bi-fold: 4 panels (front, back, inside-left, inside-right) as separate pages in projectStore
- Tri-fold / Z-fold: 6 panels as separate pages, respecting panel width ratios and reading order

### Output Schema Requirements
- Must include _isDocBackground Rect as first object per page
- Every element must have: customType, shapeKind (if shape), id (UUID), name, locked props
- Image frames: customType='image', imageId=null, fitMode='fill'
- Text frames: customType='text' with Textbox type
- Shapes: customType='shape', shapeKind='rect'|'circle'|'line'
- Color blocks: customType='colorBlock'
- All positions in px at 72dpi (mm converted via mmToPx)
- originX='left', originY='top' on all elements (Fabric.js 7 requirement)

### Integration Rules
- DO NOT modify existing stores, canvas logic, or components
- Use brandStore colors/typography if user has presets saved
- Generate proper UUIDs for all element ids (crypto.randomUUID())
- Match existing code style, patterns, and conventions

### Prompt Engineering
- System prompts explain exact Fabric.js JSON schema with examples
- Enforce valid JSON output that can be directly deserialized
- Fold-type-aware panel generation
- Photo mode: extract layout structure, colors, typography style, content zones
- Sketch mode: interpret rough drawings (boxes=content zones, scribbles=text, blobs=images)
- Retry/validation logic if AI returns malformed JSON

### Claude's Discretion
- Internal state management for the generation modal (local React state vs new store)
- Preview rendering approach (mini canvas vs static image)
- Specific prompt engineering strategies for each mode
- Error handling UX (toast vs inline errors)
- Rate limiting / request queuing approach
- Whether to stream AI responses or wait for complete JSON

</decisions>

<specifics>
## Specific Ideas

- The generation modal should be accessible from the Header (new "AI Generate" button) and optionally from the dashboard
- For photo mode, the AI should NOT copy text content — it should use placeholder text (Lorem ipsum or descriptive labels like "Headline here", "Body text")
- Brand colors from brandStore should be passed to the AI prompt when available
- Consider a "style" dropdown (minimal, bold, corporate, playful, elegant) to guide generation
- The fold type selector should show a visual diagram of each fold type

</specifics>
