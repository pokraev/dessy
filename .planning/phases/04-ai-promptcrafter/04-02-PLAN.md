    **3. Create `src/components/editor/modals/PromptCrafterModal.tsx`:**

    A modal component following the same pattern as `GenerateLeafletModal.tsx` (AnimatePresence, motion.div backdrop + panel, X close button, dark theme styling).

    **Props:** `{ open: boolean; onClose: () => void }`

    **Internal state (useState):**
    - `step: PromptStep` — starts as 'idle'
    - `baseDescription: string` — textarea value
    - `variations: PromptVariations | null`
    - `selectedVariation: 'editorial' | 'lifestyle' | 'bold' | null`
    - `customization: PromptCustomization` — all fields start as ''
    - `assembledPrompt: string` — derived from selectedVariation + customization
    - `generatedImageUrl: string | null` — base64 data URL of generated image
    - `generatedImageId: string | null` — IndexedDB key of last generated image
    - `error: string | null`
    - `frameSnapshot: FrameContext | null` — captured on modal open

    **On open (useEffect when open=true):**
    1. Read `getApiKey()` — if null, set error to i18n `promptCrafter.apiKeyMissing` and return
    2. Get `canvas` from `useCanvasStore.getState().canvasRef`
    3. Get active object: `canvas.getActiveObject()`
    4. If no active object or `customType !== 'image'`, set error to i18n `promptCrafter.noFrame`
    5. Otherwise, capture `frameSnapshot` from active object: frameId=obj.id, widthPx=obj.width*scaleX, heightPx=obj.height*scaleY, left, top, scaleX, scaleY, aspectRatio=snapAspectRatio(widthPx, heightPx), positionHint derived from position relative to canvas (e.g., "top-left", "center", "bottom-right"), brandColors from useBrandStore.getState().brandColors.map(s => s.hex), frameName=obj.name
    6. Reset step to 'idle', clear variations, selectedVariation, generatedImageUrl

    **Step 1 — idle:** Show textarea (value=baseDescription, onChange updates it), "Enrich Prompt" button (disabled if baseDescription.trim() is empty). On click: set step='enriching', call `enrichPrompt(apiKey, baseDescription, frameSnapshot)`. On success: set variations, step='customizing'. On error: set error message, step='idle'.

    **Step 2 — enriching:** Show spinner with "Crafting prompts..." text.

    **Step 3 — customizing:**
    - Show 3 variation cards (Editorial, Lifestyle, Bold) as clickable cards with the variation text preview (truncated to ~80 chars). Highlight selected with #6366f1 border. Clicking sets selectedVariation and initializes assembledPrompt.
    - Below cards: 5 customization dropdowns (mood, lighting, composition, style, background) with preset options:
      - mood: ['', 'warm', 'cool', 'dramatic', 'neutral', 'serene', 'energetic']
      - lighting: ['', 'natural daylight', 'studio', 'golden hour', 'soft diffused', 'dramatic shadow']
      - composition: ['', 'close-up', 'wide shot', 'overhead', 'eye level', 'macro detail']
      - style: ['', 'photorealistic', 'editorial', 'product', 'lifestyle', 'artistic']
      - background: ['', 'white', 'bokeh', 'solid color', 'natural', 'gradient', 'contextual']
    - Each dropdown onChange updates customization state. After any change, recompute assembledPrompt via `assemblePrompt(variations[selectedVariation], customization)`.
    - Show a textarea with assembledPrompt that is also directly editable (user can override the assembled text).
    - "Generate Image" button (disabled if no variation selected). On click: set step='generating', call `callGeminiImage(apiKey, assembledPrompt, frameSnapshot.aspectRatio)`. On success: convert base64 to blob via `base64ToBlob`, store in IndexedDB via `storeImage(blob)` to get imageId, generate thumbnail via `generateThumbnail(dataUrl)`, add to history via `addToHistory({ id: crypto.randomUUID(), imageId, thumbnailDataUrl, prompt: assembledPrompt, generatedAt: new Date().toISOString() })`, set generatedImageUrl and generatedImageId, step='result'. On error: set error, step='customizing'.

    **Step 4 — generating:** Show spinner with "Generating image..." text. Use a loading animation (e.g., motion pulse or rotate on an icon).

    **Step 5 — result:**
    - Show preview: `<img src={generatedImageUrl} />` with max-height 400px, rounded corners, centered
    - 3 buttons: "Use This" (primary #6366f1), "Regenerate" (secondary), "Edit Prompt" (secondary)
    - "Use This": Get canvas from canvasStore. Find frame by frameSnapshot.frameId via `canvas.getObjects().find(o => (o as any).id === frameSnapshot.frameId)`. If found, call `placeImageIntoFrame(canvas, frame, generatedImageUrl, generatedImageId)`, then onClose(). If frame not found, show toast via react-hot-toast with `promptCrafter.frameLost` message, still place image at canvas center using FabricImage.fromURL.
    - "Regenerate": set step='generating', re-call callGeminiImage with same assembledPrompt.
    - "Edit Prompt": set step='customizing', keep variations and selectedVariation and customization.

    **History strip (visible in customizing and result steps):**
    - Horizontal scrollable row at bottom of modal showing `usePromptCrafterStore.history` thumbnails (48x48px rounded images).
    - Label: "History" with item count.
    - Clicking a history entry: load its imageId from IndexedDB via `getImage(imageId)`, set generatedImageUrl to the object URL, set generatedImageId to entry.imageId, step='result'. Track object URLs in a ref and revoke on unmount.
    - "Clear History" small text button.
    - If empty, show "No images generated yet" text.

    **Styling:** Follow the exact dark theme pattern from GenerateLeafletModal:
    - Backdrop: fixed inset-0, bg black/60, z-50
    - Panel: bg-[#1a1a1a], border #2a2a2a, rounded-xl, max-w-2xl, max-h-[90vh] overflow-y-auto
